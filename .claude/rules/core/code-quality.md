# 代码质量要求

## 基本原则

1. **遵循现有风格** — 不引入新的编码风格，保持一致性
2. **最小化改动** — 优先编辑现有文件，只改与需求相关的代码
3. **注释原则** — 代码应自解释，只在复杂逻辑处添加说明
4. **参数化要求** — 所有方法必须参数化，禁止硬编码数据

## 安全要求

- SQL 注入：使用参数化查询
- XSS：对用户输入进行转义
- CSRF：使用 CSRF 令牌
- 敏感信息不在代码中硬编码，不在日志中输出

## 性能要求

- 避免 N+1 查询问题
- 合理使用索引
- 及时关闭资源，避免内存泄漏

## 错误处理

- 不吞掉异常
- 提供有意义的错误信息
- 区分业务异常和系统异常
- 关键操作必须记录日志
- **不为不可能发生的场景做错误处理** — 只在系统边界（用户输入、外部 API）做验证

## 提交前自查

- [ ] 没有硬编码的配置和数据
- [ ] 没有使用 localhost
- [ ] 没有安全漏洞
- [ ] 错误处理完善
- [ ] 通过所有构建检查
