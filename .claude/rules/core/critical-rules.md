# 禁止事项（CRITICAL）

> 这些规则是从实际项目中总结的血泪教训，违反会导致严重的生产问题。

## 1. 严禁硬编码 localhost 或 127.0.0.1

- 生产环境无效，跨域问题，链接失效
- **正确做法**：从配置文件或环境变量读取
- 提交前 grep 检查：`grep -r "localhost\|127\.0\.0\.1" --include="*.{代码文件扩展名}"`

## 2. 严禁使用假数据和硬编码

- 所有方法必须参数化，禁止固定值
- 测试数据通过脚本管理
- 配置项从配置文件读取

## 3. 严禁 `sshpass -p` 传递密码

- bash 双引号内特殊字符（`!@#$`）会被 shell 解释，导致密码被篡改
- **必须使用** `sshpass -f <密码文件>` 模式
- **最佳实践**：SSH 密钥认证

## 4. 严禁跳过构建验证

- 类型检查通过 ≠ 生产构建通过
- 两者必须双通过才能提交
- 不能只跑一个就认为通过

## 5. 严禁 kill 别人的进程

- 启动服务前先 `lsof -i:{port}` 确认端口空闲
- 被占就换冷门端口，不要盲目 kill
- 选端口时避开 3000/8080 等热门默认端口

## 6. 严禁在代码中硬编码敏感信息

- API Key、密码、Token 等放 `.env` 或密钥管理服务
- `.env` 必须在 `.gitignore` 中
- 提交前 `git status` 确认

## 违规处理

如果发现代码中存在上述问题：
1. **立即停止当前任务**
2. **修复所有违规代码**
3. **重新验证**
4. **追加到案例库**
