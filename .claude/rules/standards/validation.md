# 验证要求

## 强制验证清单

每次代码改动后，必须按以下顺序验证：

### 1. 构建验证
- [ ] 类型检查通过
- [ ] 生产构建通过
- 两者必须双通过，缺一不可

### 2. 后端验证
- [ ] 每个新增/修改的 API 用 curl 验证
- [ ] HTTP 状态码正确
- [ ] 返回数据格式正确

### 3. 前端冒烟
- [ ] 逐个打开所有页面，无白屏
- [ ] DevTools Console 无红色错误
- [ ] 所有枚举显示中文
- [ ] 时间格式统一

### 4. 4 层排查法
当页面数据异常时，按以下层级排查：
1. **DB** — 数据库有没有数据
2. **curl** — API 是否返回正确数据
3. **Network** — 浏览器 Network 面板状态码
4. **Component** — 前端组件渲染逻辑

任何一层不正常就是 root cause，不要只看页面有没有白屏。
