# UI/UX 规范

## 核心原则

1. **显示业务名称而非 ID** — 用户看到的必须是可理解的名称
2. **枚举值中文化** — 禁止直接展示英文枚举
3. **下拉框替代手动输入** — 值域有限的字段必须用 Select
4. **用户友好的错误信息** — 不展示技术细节给终端用户
5. **时间格式统一** — `YYYY-MM-DD HH:mm:ss`，不换行

## 表单规范

- 关联字段提供搜索选择器，禁止输入 ID
- 格式化字段（手机号/邮箱等）实时校验
- 编辑模式数据正确回填
- 所有 Select 下拉框可选择

## 列表规范

- 枚举列显示中文
- 时间列不换行
- 搜索功能正常
- 空数据有友好提示
