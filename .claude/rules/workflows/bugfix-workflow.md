# Bug 修复工作流程

## 标准流程

### 1. 问题确认
- 复现问题（curl / 浏览器 / 日志）
- 确认影响范围
- 记录现象

### 2. 根因分析
- 4 层排查法：DB → curl → Network → Component
- 不要猜测，用证据定位
- 同一问题修了 3 次还没解决 → GitHub 搜索方案

### 3. 修复实施
- 最小化改动，只修与 bug 相关的代码
- 不做额外重构
- 构建验证双通过

### 4. 验证确认
- 原问题不再复现
- 冒烟测试无新问题
- DevTools Console 无新报错

### 5. 归档记录
- commit 提交
- dev-doc 三文档
- **追加到案例库 casebook/cases**（场景/现象/原因/修复/教训）
