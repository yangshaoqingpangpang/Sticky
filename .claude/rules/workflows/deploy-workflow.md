# 部署工作流程

## 标准流程

### 1. 部署前检查
- [ ] 所有代码已提交
- [ ] 构建双通过（类型检查 + 生产构建）
- [ ] 敏感文件未被提交（.env / .secrets）
- [ ] 迁移脚本版本号无冲突

### 2. 构建
- 前端生产构建
- 后端编译
- 记录本地 commit hash

### 3. 上传部署
- SSH 连接使用 `sshpass -f <文件>` 或密钥认证
- **严禁 `sshpass -p "密码"`**
- 上传构建产物
- 执行部署脚本

### 4. 发布后验证
- curl 远端页面确认新版本
- commit hash 对比
- 品牌名回归检查
- 关键 API 冒烟测试

### 5. 回滚预案
- 部署失败时保留旧版本
- 不做破坏性回滚
- 记录失败原因

## SPA 缓存策略

| 资源 | Cache-Control | 说明 |
|------|--------------|------|
| index.html | `no-store` | 绝对不缓存 |
| hashed JS/CSS | `max-age=31536000, immutable` | 长缓存 |
