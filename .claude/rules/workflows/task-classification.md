# 指令分类与执行策略

> 接到任务前，必须先分析类型，走对应工作流。

## 任务类型

### 1. 需求设计类
- **关键词**：实现、开发、添加功能、新增
- **执行**：→ design-workflow（如有）或先设计后实现

### 2. 修复类
- **关键词**：修复、Bug、错误、不工作、白屏
- **执行**：→ bugfix-workflow

### 3. 部署类
- **关键词**：部署、发布、上线、打包
- **执行**：→ deploy-workflow

### 4. 沟通类
- **关键词**：如何、为什么、建议
- **执行**：提供分析和建议，如需代码操作转入对应分支

## 混合类型优先级

1. 部署类优先
2. 修复类次之
3. 设计类最后

## 决策流程

```
用户指令
  ↓
分析关键词和上下文
  ├─ "实现/开发/添加" → 需求设计类
  ├─ "修复/Bug/错误" → 修复类
  ├─ "部署/发布/上线" → 部署类
  └─ "如何/为什么"   → 沟通类
```
