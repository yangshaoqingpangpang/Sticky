# 反面案例索引

> 从多个项目中提炼的通用踩坑记录。写代码前必读。

### BC-001: import 方式不匹配导致白屏
- **教训**: **import 前先检查目标文件是 default export 还是 named export**

### BC-002: Dev server kill 别人的端口进程
- **教训**: **永远不要 kill 别人的端口进程，用冷门端口，被占就换**

### BC-003: Docker 端口冲突
- **教训**: **启动 Docker 前先 `docker ps` 和 `lsof -ti:{port}` 扫描端口占用**

### BC-004: 后端返回 null 数组导致前端白屏
- **教训**: **任何从后端获取的数组字段，`.map()` 前必须 `?? []` 兜底**

### BC-005: 类型检查通过 ≠ 生产构建通过
- **教训**: **类型检查和生产构建必须双通过**

### BC-006: SPA 缓存导致用户看到旧版本
- **教训**: **index.html 必须 `no-store`，hashed 资源 `immutable`**

### BC-007: dev 调试字段泄漏到生产环境
- **教训**: **调试字段必须后端 profile-gate + 前端 env-gate 双重防御**

### BC-008: 枚举大小写变体显示异常
- **教训**: **枚举映射必须覆盖所有大小写变体**

### BC-009: sshpass -p 密码含感叹号连接失败
- **教训**: **严禁 `sshpass -p`，必须用 `sshpass -f <文件>` 或 SSH 密钥**

### BC-010: 后端改路由前端 URL 没同步
- **教训**: **后端改路由必须 grep 前端消费侧 URL 同步修改**

### BC-011: 业务异常返回 HTTP 200
- **教训**: **异常处理器必须将业务错误码映射到正确的 HTTP 状态码**

### BC-012: 数据库 JSON 字段双重序列化
- **教训**: **JSON 字段返回前端时必须在 Service 层反序列化为对象**

### BC-013: 监控工具测错端口产生假告警
- **教训**: **监控必须 mirror 真实用户路径，不要直连内部端口**

### BC-014: seed 脚本数据被后台定时任务改写
- **教训**: **写 seed 前 grep 所有定时任务，确认中间态数据不会被清理**
