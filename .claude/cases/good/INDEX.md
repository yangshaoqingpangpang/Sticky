# 正面案例索引

> 经过验证的成功模式，可直接复用。

### GC-001: Provider 模式抽象服务商
- **模式**: 定义接口 + 具体实现类
- **复用**: 对接多个第三方服务时

### GC-002: 环境变量驱动的配置
- **模式**: 配置文件全部用 `${ENV_VAR:default}`
- **复用**: 所有项目的标准做法

### GC-003: 幂等的数据库迁移脚本
- **模式**: 先检查再操作（IF NOT EXISTS / information_schema）
- **复用**: 所有增量 DDL 改动

### GC-004: 并行 Agent 加速大型任务
- **模式**: 独立子任务分配给多个后台 agent 并行执行
- **复用**: 可分解为独立子任务的大型工程

### GC-005: 事件驱动异步处理
- **模式**: 主操作快速返回 + 副作用异步执行
- **复用**: 任何"主操作不能被副作用阻塞"的场景

### GC-006: sshpass -f 文件模式
- **模式**: 密码写入临时文件 → `sshpass -f <文件>` → 用完删除
- **复用**: 所有需要 sshpass 的自动化脚本
