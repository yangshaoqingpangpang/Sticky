# 案例库系统

> 从实际项目中积累的经验教训，分为反面案例（bad）和正面案例（good）。

## 目录结构

```
cases/
├── README.md          # 本文件
├── bad/
│   └── INDEX.md       # 反面案例索引
└── good/
    └── INDEX.md       # 正面案例索引
```

## 使用方式

1. **写代码前**：浏览 `bad/INDEX.md`，检查是否有相关的踩坑记录
2. **遇到新问题**：修复后立即添加到 `bad/INDEX.md`
3. **发现好模式**：添加到 `good/INDEX.md`

## 记录格式

### 反面案例
```markdown
### BC-XXX: {简短标题}
- **场景**: {什么情况下触发}
- **现象**: {用户看到什么}
- **原因**: {根本原因}
- **修复**: {如何修复}
- **教训**: **{加粗的通用教训}**
```

### 正面案例
```markdown
### GC-XXX: {简短标题}
- **模式**: {使用了什么模式}
- **效果**: {带来什么好处}
- **复用**: {什么场景可以复用}
```
